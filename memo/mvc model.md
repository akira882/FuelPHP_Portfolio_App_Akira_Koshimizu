# 第1章：MVCモデル（技術的定義）
MVCモデルとは、アプリケーションの処理を「データ処理」「表示」「制御」の3つの役割に分割して実装するアーキテクチャ（設計手法）です。
ファイルが分かれることで、保守性が高まります。

1. Model（モデル） = データロジック担当
定義: データベースとの通信、データの登録・取得・削除、およびデータの加工ロジックを担当するプログラム群です。
役割: Controllerから指示を受け、SQLを発行してデータベースを操作し、その結果（データ）をControllerに返します。
ファイル場所: fuel/app/classes/model/

2. View（ビュー） = ユーザーインターフェース担当
定義: ユーザーがブラウザで見る画面（HTML/CSS）を生成するプログラム群です。
役割: Controllerから渡された変数（データ）を、HTMLの中に埋め込んで表示します。ロジック（複雑な計算など）は記述しません。
ファイル場所: fuel/app/views/

3. Controller（コントローラー） = 制御担当
定義: ユーザーのリクエスト（URLアクセスやフォーム送信）を受け取り、処理の流れを制御するプログラム群です。
役割:
Modelを呼び出してデータを取得・保存させる。
そのデータをViewに渡す。
最終的な画面をブラウザに返す（レスポンス）。
ファイル場所: fuel/app/classes/controller/
